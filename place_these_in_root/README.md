# Starter Root Files

Copy these files into the root of a new target app repo before the first `autoresearch` run.

- `ABOUT.md` should say what the app is, who it is for, and what problem it solves.
- `BRAND.md` should say how the app should feel and present itself.
- `FEATURES.md` should say what the app can currently do.
- `REFLECTION.md` starts the short failure-memory file in the right format.
- `TODO.md` starts the review-debt checklist the harness uses after kept changes.
- `results.tsv` starts the experiment log with the right header row.

Keep `results.tsv`, `REFLECTION.md`, `TODO.md`, and `.autoresearch-cache/` untracked in the target repo.
